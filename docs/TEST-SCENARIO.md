# Hamidun-Setup — полный сценарий ручного тестирования

**Версия:** 2026-07-18 · **Что тестируем:** лендинг → Telegram-бот → скачивание → установка → пост-установка → бот-сопровождение. Вся петля глазами нового пользователя.
**Время:** ~2–3 часа на платформу (не считая ожиданий догревов).

---

## 0. Подготовка

**Окружения (матрица):**

| # | Платформа | Что нужно |
|---|-----------|-----------|
| 1 | Windows 10 или 11, «чистая» (VM/второй ПК, без Node/Git/VS Code) | права администратора |
| 2 | macOS Apple Silicon (M1–M4) | ничего дополнительно |
| 3 | macOS Intel (2015–2020) | ничего дополнительно |
| 4 | Telegram: телефон + десктоп-клиент | **свежий ТГ-аккаунт**, который никогда не писал боту |
| 5 | Браузеры: Chrome + Safari (на Mac), мобильный браузер на телефоне | — |

**Правила фиксации:** на каждый дефект — скриншот + номер шага + «ожидал / получил». Шаблон бага в конце документа. Время выполнения шагов с таймерами (догревы) фиксируй по часам.

**Известные особенности (НЕ баги, но проверь, что они выглядят приемлемо):**
- Установщики большие: Windows ~2.0 ГБ, Mac ~2.7 ГБ.
- Крупные компоненты во время установки могут «молчать» несколько минут — прогресс не движется, это норма (но заметь, если дольше ~10 мин).
- Windows-exe не подписан → SmartScreen обязателен. Мы его объясняем, а не обходим.

## Блок D — Установка Windows

На чистой Windows 10/11. Скачанный `Hamidun-Setup-Windows.exe`.

| Шаг | Действие | Ожидание | ✔/✘ |
|-----|----------|----------|------|
| D1 | Запусти exe | Синее окно SmartScreen → «Подробнее» → «Выполнить в любом случае» — ровно как обещали лендинг и бот | |
| D2 | UAC-запрос прав администратора | Есть, издатель/имя не пугают сверх ожидаемого | |
| D3 | Иконка exe в проводнике | Логотип Hamidun (синий бейдж с H), не дефолтный Electron | |
| D4 | Экран выбора компонентов | Дефолты: VS Code, Git, Node, Python, Claude Code, расширение, конфиг, курс — ВКЛ; Cursor и Nomad — как задумано; описания понятны новичку; паки скиллов листаются | |
| D5 | Ничего не меняя — «Установить» | Прогресс идёт, журнал живой; советы-карусель говорят про VS Code (не Cursor) | |
| D6 | Засеки общее время | ~15–60 мин в зависимости от машины; «молчание» на крупных компонентах ≤ ~10 мин | |
| D7 | Финальный экран | «Готово» + 3 шага; ОБЕ карточки подключения нейросети: «Своя подписка Claude» И «Из России — через Nomad» (вторая — даже если Nomad не выбирал, с корректным текстом «можно доустановить») | |
| D8 | Кнопка «Открыть VS Code» | VS Code открывается, слева — панель Claude (иконка ✳) | |
| D9 | Памятка START-HERE | Открылась/доступна; говорит «открой VS Code» (Cursor — только как опция) | |
| D10 | ОФЛАЙН-тест (отдельный прогон): отключи интернет ПОСЛЕ скачивания, запусти установку с дефолтами | Всё ставится офлайн без падений (кроме явно онлайновых шагов) | |
| D11 | Повторный запуск установщика после успешной установки | Видит уже установленное, не ломает; можно доотметить один компонент (например Nomad) и поставить только его | |
| D12 | Прерви установку на середине (закрой окно), запусти заново | Восстанавливается без загадочных ошибок | |

**Проверка результата (терминал VS Code):** `git --version`, `node -v`, `python --version`, `claude --version` — все отвечают. Шрифт JetBrains Mono применился в редакторе.

## Шаблон бага

```
[Платформа/блок/шаг]: Win11 / D7
Ожидал: обе карточки подключения нейросети
Получил: только карточка Claude
Скриншот: ...
Время: 18.07 17:42
Воспроизводится: да/нет/через раз
```

**Критерий приёмки всей системы:** новичок без посторонней помощи проходит путь «увидел лендинг → скачал → установил → подключил ИИ (любым из двух путей) → получил первый результат», а на каждом затыке бот реально помогает в течение минуты диалога.

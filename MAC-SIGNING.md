# Подпись и нотаризация macOS (убрать «приложение повреждено»)

Пока приложение не подписано сертификатом Apple, macOS показывает «повреждено».
Полностью убирает это только **Apple Developer Program ($99/год) + подпись Developer ID + нотаризация**.
Этот файл — что делает Жемал (вручную, иначе никак) и что автоматизирую я после.

## Почему я НЕ могу зарегистрировать аккаунт за тебя

Проверено на актуальном флоу Apple (2026). Регистрация физически требует тебя:
1. **Вход в твой Apple ID + 2FA** — код приходит на твоё доверенное устройство, перехватить нельзя.
2. **Юридическое согласие** — принятие Apple Developer Agreement от твоего имени (обязывающее действие).
3. **Твоя карта** — оплата $99 только твоей личной картой (иначе задержка + запрос гос-ID).
4. **Проверка личности** — Apple сверяет ЛЕГАЛЬНОЕ имя (в профиле Apple ID — твоё настоящее имя, не ник/компания), может попросить фото гос-ID.
5. Автоматизировать вход в Apple ID через браузер = риск блокировки аккаунта (антибот).

Итог: это ~10 минут ручной работы, которую можешь сделать только ты. Ниже — точные шаги.

## Шаги (тебе, ~10 мин)

1. Убедись, что в твоём **Apple ID включена 2FA** и в профиле стоит **настоящее имя** (First/Last — латиницей как в документах; ник/название компании → задержка одобрения).
2. Открой **https://developer.apple.com/programs/enroll/** → войди своим Apple ID.
   Либо приложение **Apple Developer** на iPhone/iPad (в РФ/Бразилии часто быстрее с identity-проверкой).
3. Тип: **Individual / Sole Proprietor** (для личного бренда — не Organization, там нужен D-U-N-S).
4. Подтверди имя/почту/телефон/адрес, прими соглашение, оплати **$99** своей картой.
5. Дождись одобрения (обычно от пары часов до пары суток; иногда просят фото гос-ID).

## Что прислать мне после одобрения (я подключу CI за ~30 мин)

Через **GitHub → Settings → Secrets and variables → Actions** (repo JHamidun/hamidun-installer),
секреты кладутся зашифрованно, я их не вижу в открытом виде:

1. **Developer ID Application** сертификат, экспортированный из Keychain в `.p12`:
   - секрет `CSC_LINK` = base64 от .p12 (`base64 -i cert.p12 | pbcopy`);
   - секрет `CSC_KEY_PASSWORD` = пароль от .p12.
2. **App Store Connect API key** (для нотаризации, App Store Connect → Users and Access → Integrations → Keys):
   - секрет `APPLE_API_KEY` = содержимое файла `AuthKey_XXXX.p8` (или base64);
   - секрет `APPLE_API_KEY_ID` = Key ID;
   - секрет `APPLE_API_ISSUER` = Issuer ID.
3. Твой **Team ID** (10 символов, виден в developer.apple.com → Membership).

## Что делаю я, когда секреты появятся

1. В `package.json` → `mac`: включаю `hardenedRuntime: true`, `entitlements`/`entitlementsInherit`
   = `build/entitlements.mac.plist` (уже лежит), `notarize` по Team ID; `afterPack` ad-hoc-хук
   отключается автоматически (подписывает уже настоящий Developer ID).
2. В `.github/workflows/build-mac.yml`: убираю `CSC_IDENTITY_AUTO_DISCOVERY=false`, пробрасываю
   секреты в окружение — electron-builder сам подпишет **Developer ID** + нотаризует + сделает staple.
3. Результат: скачанный dmg открывается **как обычная программа** — без «повреждено», без `xattr`.

## Пока аккаунта нет (текущее состояние)

Сборка идёт с **ad-hoc подписью** (`tools/mac-adhoc-sign.js`) + инструкцией внутри dmg. Пользователь
открывает так: перетащить в «Программы», затем в Терминале `xattr -cr "/Applications/Hamidun Setup.app"`.
Рабочий временный вариант; нотаризация убирает и этот шаг.
